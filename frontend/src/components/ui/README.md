# UI Components

Primitive, reusable UI components that form the design system foundation.

## Contains

- **Button** - Primary, secondary, ghost, and icon button variants
- **Input** - Text inputs, password fields, file inputs
- **Card** - Content containers with various styles
- **Modal** - Overlay dialogs and confirmation prompts
- **Badge** - Status indicators and labels
- **Avatar** - User profile images
- **Spinner** - Loading indicators
- **Toast** - Notification messages
- **Dropdown** - Select menus and action dropdowns

## Guidelines

- Components should be purely presentational
- Accept styling via props (variant, size, etc.)
- No business logic or API calls
- Fully accessible (ARIA attributes, keyboard navigation)
